# `/dev` 完整编辑器交接说明

## 入口与安全边界

编辑器入口是 `https://g.anyq.site/dev`（本地为 `http://127.0.0.1:9500/dev`）。它是独立 HTML 页面，不加载正式游戏的 `app.js`，也不创建 WebSocket 连接。所有修改都停留在浏览器内存，导出前不会写入 `src/data`、`public/data` 或服务器。

## 当前能力

- 左侧赛道/图层树和 MP03 → MP04 → MP02 → MP03 → MP04 → MP02 分段导航。
- 中央 Canvas 支持网格、滚轮缩放、中键/右键拖拽平移；碰撞层可以画格、擦格、选中格。
- 出生点层可点选并编辑 X/Y、重力和起始速度；终点 X、淘汰线余量及上下边界可在属性检查器修改，并纳入撤销历史。画布会显示镜头目标、终点和淘汰线参考线。
- 装饰层会加载三张原始 MP 视觉图集的前后景放置；点击“装饰层”后可拖动已有装饰，松开即写入可撤销草稿并在测试包中标记 `visualsChanged`。
- 撤销/重做使用不可变草稿历史；marathon 原始数据里的重复碰撞格会在编辑器入口归一化。
- `运行`、`暂停`、`单帧`、`翻转` 使用同一个 `GameRoom` 物理核心；不会复制另一套碰撞实现。
- 角色与物理检查器会显示实时碰撞盒、重力、速度、静态接触格和 32 帧预测轨迹；碰撞宽高、重力倍率、追赶倍率可以应用后立即单帧验证。
- 翻转录制与确定性回放：事件按 40 FPS tick 保存，回放不依赖随机数。
- 动画检查器可以选择四个角色和跑步/翻转/下落/出生/淘汰状态，直接编辑帧序列和播放速度，显示原始 15×9 图集帧。
- 本地实验室可以模拟 1–4 个玩家，并对显示快照注入 0–300ms 延迟与 0–80% 确定性丢包；权威物理仍在本地运行。
- 时间轴下方保留最近的水平碰撞、抵达终点和淘汰事件，便于把“卡墙/穿墙/淘汰”复现到具体 tick。
- 画布下方的独立时间轴面板支持回到起点、跳到当前和按 tick 拖动重播定位；右侧错误栏只显示编辑器校验/加载错误。
- 导出草稿、导入草稿；导出/导入测试包。测试包包括草稿、碰撞/装饰 diff、实验参数、物理参数、动画帧配置和翻转回放。

## 建议工作流

1. 打开 `/dev`，先从左侧选择赛段，按中键拖动或滚轮缩放到目标区域。
2. 画/擦碰撞格，观察右侧验证信息和选中坐标；必要时点“撤销/重做”。
3. 点击“重置模拟”，选择本地玩家数量、延迟、丢包，然后用“单帧/翻转/录制”稳定复现问题。
4. 点击“提交测试包”。这只是下载 JSON，不是发布；把 JSON 交给另一位 AI 或开发者审核。
5. 在项目根目录执行：

   ```powershell
   npm run validate:editor-package -- .\gswitch-test-package.json
   npm test
   npm run test:coverage
   ```

6. 审核 diff、回放截图和自动测试结果后，才把经过确认的碰撞数据更新到 `src/data`，重新运行 `npm run build:data`，再走正常 Git review 和部署流程。

## 代码分层

| 文件 | 责任 |
|---|---|
| `public/dev.html` / `editor.css` | 编辑器独立布局 |
| `public/editor.js` | 画布交互、GameRoom 调试循环、动画预览和导入导出 |
| `public/editor-draft.js` | 草稿模型、碰撞画擦、撤销重做、校验 |
| `public/editor-inspector.js` | 碰撞盒、静态接触检测和本地预测轨迹的纯函数 |
| `public/editor-replay.js` | 40 FPS 翻转回放和测试包格式 |
| `public/editor-network.js` | 本地延迟/丢包显示实验，不连接生产 WebSocket |
| `public/editor-package.js` | 生成和校验碰撞 diff |
| `tools/validate-editor-package.mjs` | CI/交接前的本地测试包校验 |

## 明确未完成的生产级能力

装饰资源的新增/删除、多人 bot 行为脚本和服务器端权限仍未接入；当前 `/dev` 可移动并导出已有装饰，仍不会把未经审核的资源写入生产赛道。新增能力应继续在草稿模型上实现，不能让编辑器直接写生产文件或开放成公网管理后台。
